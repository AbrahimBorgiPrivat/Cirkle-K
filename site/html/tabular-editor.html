Tabular Editor And Reproducible Measures

Both PBIP workspaces now include Tabular assets so measures are not only stored inside the model, but also as readable source files.

Reproducibility focus

  • Measure logic is stored as text, so it can be reviewed and versioned like code.
  • Logical groups and display folders are recreated through scripts instead of manual clicks.
  • The same setup can be rerun to keep the semantic model aligned with source control.

How it is structured

  • Each workspace has a local `Tabular` folder.
  • Each measure is stored as its own `.dax` file.
  • Each logical group has a `.csx` script that reads the `.dax` files and recreates the measures.
  • A shared `TabularEditorCLITool` C# library provides helper builders used by the scripts.

Why this improves reproducibility

  • Measure definitions are diff-friendly and code-review friendly.
  • Folder structure and display folders are defined in source, not only in the PBIP model state.
  • The same script can be rerun after model refreshes or workspace changes.
  • The helper DLL makes the scripting approach consistent across both workspaces.

Execution flow

  • `run_tabular_scripts.ps1` builds the helper library if needed.
  • Tabular Editor 2 runs the workspace-local `.csx` scripts.
  • The scripts recreate or update measures from the `.dax` files.
  • This keeps the semantic model aligned with the checked-in source assets.
Used here

The interview workspace has grouped KPI and visual measures. The simulated workspace groups revenue, activity, loyalty, and campaign measures.
